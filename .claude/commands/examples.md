## Implementation Strategy

### Documentation Integration

Each example should be:

1. **Executable Script** (`examples/category/example-name.ts`)
   - Uses `// SNIPPET:START` and `// SNIPPET:END` markers
   - Includes test assertions below snippet
   - Exits 0 on success, non-zero on failure
   - Uses relative imports for execution: `../../src/...`

2. **Generated MDX Doc** (`docs/examples/category/example-name.mdx`)
   - Auto-generated by `scripts/generate-example-docs.ts`
   - Extracts snippet between markers
   - Transforms imports to package style: `@tevm/voltaire/Module`
   - Includes frontmatter (title, description)
   - Links to source file
   - Searchable in docs

3. **Sidebar Navigation** (`docs/mint.json`)
   - Manually add to navigation array
   - Nested under "Examples" group
   - Organized by category (Getting Started, Addresses, etc.)
   - Enables search/discovery in docs
   - MCP server will index these pages

### Why This Matters

- **Searchable**: Examples appear in docs search
- **Discoverable**: MCP server indexes example pages
- **Tested**: All examples are executable and validated
- **Accurate**: Auto-generated from working code
- **Maintainable**: Single source of truth (the executable script)

### Workflow

```bash
# 1. Create example script
vim examples/hashing/keccak256-string.ts

# 2. Test it
bun run examples/hashing/keccak256-string.ts

# 3. Generate MDX
bun run examples:generate

# 4. Add to docs/mint.json navigation
# (Manual step - add to Examples group)

# 5. Verify in docs
bun run docs:dev
```
