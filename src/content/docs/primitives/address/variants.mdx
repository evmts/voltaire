# Address Variants

Branded hex string types representing different address formats with compile-time guarantees.

## Overview

Address variants provide type-safe hex strings with specific casing guarantees:
- **Checksummed** - EIP-55 mixed-case format for display and verification
- **Lowercase** - All lowercase hex for canonical storage
- **Uppercase** - All uppercase hex for specific protocols

Each variant is a [branded](/primitives/branded-types) hex string type (`string & { __tag, __variant, __casing }`), not a `Uint8Array`.

## ChecksumAddress

EIP-55 checksummed address format with mixed casing based on keccak256 hash.

### `Address.Checksummed.from(addr)`

Create checksummed hex string from Address.

```typescript
const addr = Address.fromHex("0x742d35cc6634c0532925a3b844bc9e7595f51e3e")

// Get checksummed string
const checksummed = Address.Checksummed.from(addr)
// "0x742d35Cc6634C0532925a3b844Bc9e7595f51e3e" (mixed case)

// Type is branded Checksummed, not BrandedAddress
typeof checksummed // "string"
```

## Relationship to Hex Module

All address variants are built on `Hex.Sizedless than 20>`, a 20-byte hex string type:

```typescript
// Structure of each variant
type Checksummed = Hex.Sizedless than 20> & { __checksummed: true }
type Lowercase = Hex.Sizedless than 20> & { __lowercase: true }
type Uppercase = Hex.Sizedless than 20> & { __uppercase: true }
```

These are return types from variant methods, not separate exports. Use `Address.Checksummed.from()`, `Address.Lowercase.from()`, and `Address.Uppercase.from()` to create them.

**Parameters:**
- `addr: BrandedAddress` - Address to checksum

**Returns:** `Checksummed` - Branded checksummed hex string

**Type:**
```typescript
type Checksummed = string & {
  readonly __tag: 'Hex'
  readonly __variant: 'Address'
  readonly __checksummed: true
}
```

### `Address.Checksummed.isValid(str)`

Validate EIP-55 checksum of address string.

```typescript
// Valid checksummed
Address.Checksummed.isValid("0x742d35Cc6634C0532925a3b844Bc9e7595f51e3e") // true

// Invalid checksum (wrong casing)
Address.Checksummed.isValid("0x742d35cc6634c0532925a3b844bc9e7595f51e3e") // false
Address.Checksummed.isValid("0x742D35CC6634C0532925A3B844BC9E7595F51E3E") // false

// All lowercase/uppercase passes (no mixed case = no checksum)
Address.Checksummed.isValid("0x742d35cc6634c0532925a3b844bc9e7595f51e3e") // true
Address.Checksummed.isValid("0x742D35CC6634C0532925A3B844BC9E7595F51E3E") // true
```

**Parameters:**
- `str: string` - Address string to validate

**Returns:** `boolean`

Defined in: [primitives/Address/BrandedAddress/ChecksumAddress.js:55](https://github.com/evmts/voltaire/blob/main/src/primitives/Address/BrandedAddress/ChecksumAddress.js)
Tests: [ChecksumAddress.test.ts](https://github.com/evmts/voltaire/blob/main/src/primitives/Address/BrandedAddress/ChecksumAddress.test.ts)

**EIP-55 Logic:**
- If all letters same case (all lower or all upper), valid
- If mixed case, each letter's case must match keccak256-based checksum

## Usage Patterns

### Display vs Storage

```typescript
const addr = Address.fromHex("0x742d35cc6634c0532925a3b844bc9e7595f51e3e")

// For display to users (with checksum)
const display = Address.Checksummed.from(addr)
console.log(`Address: ${display}`) // "0x742d35Cc..."

// For canonical storage (lowercase)
const storage = Address.Lowercase.from(addr)
database.save({ address: storage }) // "0x742d35cc..."

// For specific protocols requiring uppercase
const uppercase = Address.Uppercase.from(addr)
```

### Validating User Input

```typescript
function acceptAddress(input: string): Address {
  // Require valid checksum from user
  if (!Address.Checksummed.isValid(input)) {
    throw new Error("Invalid address checksum")
  }
  return Address.fromHex(input)
}

// ✓ Valid checksummed
acceptAddress("0x742d35Cc6634C0532925a3b844Bc9e7595f51e3e")

// ✗ Invalid checksum
try {
  acceptAddress("0x742d35cc6634c0532925a3b844bc9e7595f51e3e")
} catch (e) {
  console.error(e.message) // "Invalid address checksum"
}
```

### Type-Safe APIs

```typescript
interface Transaction {
  to: Lowercase  // Always store as lowercase
}

interface DisplayConfig {
  address: Checksummed  // Always display with checksum
}

const addr = Address.fromHex("0x742d35Cc6634C0532925a3b844Bc9e7595f51e3e")

const tx: Transaction = {
  to: Address.Lowercase.from(addr)  // ✓ type-safe
}

const config: DisplayConfig = {
  address: Address.Checksummed.from(addr)  // ✓ type-safe
}
```

## Variant Comparison

| Variant | Casing | Use Case | Keccak256 Needed |
|---------|--------|----------|------------------|
| **Checksummed** | Mixed | Display, verification | Yes |
| **Lowercase** | All lower | Canonical storage, comparison | No |
| **Uppercase** | All upper | Specific protocols | No |

## Instance Methods

Address class provides convenient instance methods that return variants:

```typescript
const addr = new Address("0x742d35Cc6634C0532925a3b844Bc9e7595f51e3e")

// Instance methods
addr.toChecksummed()  // Checksummed
addr.toLowercase()    // Lowercase (alias for toHex())
addr.toUppercase()    // Uppercase

// Equivalent to
Address.Checksummed.from(addr)
Address.Lowercase.from(addr)
Address.Uppercase.from(addr)
```

See [Conversions](/primitives/address/conversions) for details on instance methods.

## Tree-Shakeable Usage

All variants available as tree-shakeable namespace imports:

```typescript
import * as ChecksumAddress from '@tevm/voltaire/BrandedAddress/ChecksumAddress'
import * as LowercaseAddress from '@tevm/voltaire/BrandedAddress/LowercaseAddress'
import * as UppercaseAddress from '@tevm/voltaire/BrandedAddress/UppercaseAddress'
import type { BrandedAddress } from '@tevm/voltaire/BrandedAddress'

const addr: BrandedAddress = /* ... */

const checksummed = ChecksumAddress.from(addr)
const lowercase = LowercaseAddress.from(addr)
const uppercase = UppercaseAddress.from(addr)

// Validation
ChecksumAddress.isValid("0x742d35Cc6634C0532925a3b844Bc9e7595f51e3e")
```

**Tree-shaking benefit:** When avoiding `ChecksumAddress`, keccak256 implementation is excluded from bundle. Lowercase and Uppercase variants have no crypto dependencies.

See [BrandedAddress](/primitives/address/branded-address) for details on tree-shakeable API.
