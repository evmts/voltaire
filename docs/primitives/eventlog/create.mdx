## `EventLog.create(params: EventLogParams): EventLog`

Explicit constructor for event log. Alias for `EventLog()` constructor.

**Parameters:**
- `params.address: BrandedAddress` - Contract address that emitted the log
- `params.topics: readonly BrandedHash[]` - Topic hashes
- `params.data: Uint8Array` - Non-indexed event data
- `params.blockNumber?: bigint` - Block number
- `params.blockHash?: BrandedHash` - Block hash
- `params.transactionHash?: BrandedHash` - Transaction hash
- `params.transactionIndex?: number` - Transaction index
- `params.logIndex?: number` - Log index
- `params.removed?: boolean` - Removed flag

**Returns:** `EventLog` - Event log instance with prototype methods

**Example:**

```typescript
import { EventLog, Address, Hash } from '@tevm/voltaire';

const log = EventLog.create({
  address: Address.fromHex('0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb2'),
  topics: [Hash.fromHex('0xddf252ad...')],
  data: new Uint8Array([0, 0, 0, 1]),
  blockNumber: 18000000n,
});

console.log(log.blockNumber); // 18000000n
```

## Usage Notes

`create()` is functionally identical to `EventLog()` and `EventLog.from()`. Use whichever name is clearest:

```typescript
// All equivalent
const log1 = EventLog(params);       // Primary (recommended)
const log2 = EventLog.from(params);  // Alias
const log3 = EventLog.create(params); // Alias
```

Most code uses `EventLog()` as the primary constructor. Use `create()` when explicit construction intent improves readability.
