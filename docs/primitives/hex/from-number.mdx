## Number Range

Valid input range: `0` to `Number.MAX_SAFE_INTEGER` (2^53 - 1)

```typescript
// Valid
Hex(0)                      // "0x0"
Hex(Number.MAX_SAFE_INTEGER)  // "0x1fffffffffffff"

// For larger values, use fromBigInt
const large = 2n ** 256n - 1n
Hex.fromBigInt(large, 32)  // Max uint256
```

## Size Padding

The `size` parameter specifies the **byte length** of the output:

```typescript
// No padding (minimal representation)
Hex(255)      // "0xff" (1 byte)

// 2-byte padding
Hex(255, 2)   // "0x00ff"

// 4-byte padding (uint32)
Hex(255, 4)   // "0x000000ff"

// 32-byte padding (uint256)
Hex(255, 32)  // "0x00...ff" (64 hex chars)
```

## Common Use Cases

**Transaction Values:**
```typescript
const gasPrice = 20_000_000_000  // 20 gwei
const hex = Hex(gasPrice)  // "0x4a817c800"
```

**Block Numbers:**
```typescript
const blockNumber = 18_000_000
const hex = Hex(blockNumber)  // "0x112a880"
```

**Fixed-Size Fields:**
```typescript
// Chain ID (uint256)
const chainId = 1  // Ethereum mainnet
const hex = Hex(chainId, 32)  // 32-byte padded
```

**Array Indices:**
```typescript
const index = 42
const hex = Hex(index, 32)  // For storage slot calculation
```

## Performance

Direct `toString(16)` conversion with optional `padStart`. Very fast for small numbers.

For values requiring padding, consider pre-allocating buffers in performance-critical code.
